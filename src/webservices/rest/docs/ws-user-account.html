WS User Account Service Documentation

The WS User Account Service allows you to manage user accounts through multiple HTTP methods: creating, retrieving, updating, and deleting accounts.

Note for Beginners: This guide provides easy-to-follow examples for each HTTP method using both Burp Repeater and curl.

Endpoint

/webservices/rest/ws-user-account.php

Supported HTTP Methods

  • GET: Retrieve account information.
  • POST: Create a new account.
  • PUT: Update or create an account.
  • DELETE: Delete an existing account.
  • OPTIONS: Retrieve CORS-related metadata.

Request and Response Examples

1. GET (Retrieve Account Information)

GET /webservices/rest/ws-user-account.php?username=adrian HTTP/1.1
Host: mutillidae.localhost
Accept: application/json
Connection: close

Response

{
    "Result": {
        "Accounts": [
            {
                "username": "adrian",
                "firstname": "Adrian",
                "lastname": "Smith",
                "signature": "AS"
            }
        ]
    }
}

2. POST (Create a New Account)

POST /webservices/rest/ws-user-account.php HTTP/1.1
Host: mutillidae.localhost
Content-Type: application/x-www-form-urlencoded

username=john&password=pass123&firstname=John&lastname=Doe&signature=JD

Response

{
    "Result": "Inserted account 'john'",
    "Success": true
}

3. PUT (Update or Create an Account)

If the account exists, it will be updated; otherwise, it will be created.

PUT /webservices/rest/ws-user-account.php HTTP/1.1
Host: mutillidae.localhost
Content-Type: application/x-www-form-urlencoded

username=john&password=newpass123&firstname=John&lastname=Doe&signature=JDoeUpdated

Response

{
    "Result": "Updated account 'john'.",
    "RowsAffected": 1,
    "Success": true
}

4. DELETE (Delete an Account)

DELETE /webservices/rest/ws-user-account.php HTTP/1.1
Host: mutillidae.localhost
Content-Type: application/x-www-form-urlencoded

username=john&password=newpass123

Response

{
    "Result": "Deleted account 'john'.",
    "Success": true
}

5. OPTIONS (CORS Metadata)

OPTIONS /webservices/rest/ws-user-account.php HTTP/1.1
Host: mutillidae.localhost
Origin: http://mutillidae.localhost

Response

HTTP/1.1 204 No Content
Access-Control-Allow-Origin: *
Access-Control-Allow-Methods: GET, POST, PUT, DELETE, OPTIONS
Access-Control-Allow-Headers: Content-Type, Authorization
Troubleshooting Tips:
  • Ensure that the mutillidae.localhost host is correctly configured in your environment.
  • If using Burp Suite, confirm that your browser or tool is correctly routing traffic.
  • Review CORS headers in the OPTIONS response to avoid cross-origin issues.
