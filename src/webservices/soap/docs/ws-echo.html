SOAP Echo Service Documentation

The SOAP Echo Service receives an input message and sends back the same message. Itâ€™s a useful tool to verify if your request was processed correctly.

Endpoint

POST /soap/ws-echo.php

Request Structure

Below is an example of the SOAP request body.

<soapenv:Envelope xmlns:soapenv="http://schemas.xmlsoap.org/soap/envelope/">
   <soapenv:Header/>
   <soapenv:Body>
      <echo>
         <message>Hello SOAP World!</message>
      </echo>
   </soapenv:Body>
</soapenv:Envelope>

Step-by-Step Example Using Burp Repeater

POST /soap/ws-echo.php HTTP/1.1
Host: mutillidae.localhost
Content-Type: text/xml; charset=utf-8
SOAPAction: "urn:echo"

Expected Response

<?xml version="1.0"?>
<soap:Envelope xmlns:soap="http://schemas.xmlsoap.org/soap/envelope/">
   <soap:Body>
      <echoResponse>
         <message>Hello SOAP World!</message>
      </echoResponse>
   </soap:Body>
</soap:Envelope>
Troubleshooting Tips:
  • Ensure that the service is accessible at mutillidae.localhost.
  • If using Burp Repeater, make sure the Content-Type header is set correctly.
  • Use the correct SOAPAction header for SOAP requests.
