SOAP User Account Service Documentation

This service provides SOAP-based operations for managing user accounts. The available operations include getting user information, registering new users, updating existing accounts, and deleting users.

Endpoint

POST /webservices/soap/ws-user-account.php

Available Operations

  • getUser: Fetches user information.
  • registerUser: Creates a new user account.
  • updateUser: Updates an existing account or creates it if it doesnâ€™t exist.
  • deleteUser: Deletes an existing user account.

Example Request and Response

1. getUser Example

POST /webservices/soap/ws-user-account.php HTTP/1.1
Host: mutillidae.localhost
Content-Type: text/xml; charset=UTF-8
SOAPAction: "urn:ws-user-account#getUser"

<soapenv:Envelope xmlns:soapenv="http://schemas.xmlsoap.org/soap/envelope/"
                  xmlns:urn="urn:ws-user-account">
   <soapenv:Header/>
   <soapenv:Body>
      <urn:getUser>
         <username>Jeremy</username>
      </urn:getUser>
   </soapenv:Body>
</soapenv:Envelope>

Expected Response:

<accounts message="Results for Jeremy">
   <account>
      <username>Jeremy</username>
      <firstname>Jeremy</firstname>
      <lastname>Doe</lastname>
   </account>
</accounts>

2. registerUser Example

POST /webservices/soap/ws-user-account.php HTTP/1.1
Host: mutillidae.localhost
Content-Type: text/xml; charset=UTF-8
SOAPAction: "urn:ws-user-account#registerUser"

<soapenv:Envelope xmlns:soapenv="http://schemas.xmlsoap.org/soap/envelope/"
                  xmlns:urn="urn:ws-user-account">
   <soapenv:Header/>
   <soapenv:Body>
      <urn:registerUser>
         <username>Joe</username>
         <password>Holly</password>
         <firstname>Joe</firstname>
         <lastname>Holly</lastname>
         <signature>Try Harder</signature>
      </urn:registerUser>
   </soapenv:Body>
</soapenv:Envelope>

Expected Response:

<accounts message="Inserted account Joe" />

Troubleshooting Tips:
  • Ensure that the mutillidae.localhost host is correctly configured in your environment.
  • If using Burp Repeater, set the correct SOAPAction header.
  • Ensure the Content-Type header is text/xml; charset=UTF-8.
