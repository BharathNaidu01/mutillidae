SOAP User Account Service Documentation

This service offers SOAP-based operations for user account management, including retrieving, creating, updating, and deleting user accounts.

Endpoint

POST /webservices/soap/ws-user-account.php

Operations

  • getUser: Fetches user information.
  • registerUser: Creates a new user account.
  • updateUser: Updates an existing user or creates it if it doesn't exist.
  • deleteUser: Deletes an existing user account.

1. getUser Operation

POST /webservices/soap/ws-user-account.php HTTP/1.1
Host: mutillidae.localhost
Content-Type: text/xml; charset=UTF-8
SOAPAction: "urn:ws-user-account#getUser"

<soapenv:Envelope xmlns:soapenv="http://schemas.xmlsoap.org/soap/envelope/" 
                  xmlns:urn="urn:ws-user-account">
   <soapenv:Header/>
   <soapenv:Body>
      <urn:getUser>
         <username>Jeremy</username>
      </urn:getUser>
   </soapenv:Body>
</soapenv:Envelope>

cURL Command

curl -X POST "http://mutillidae.localhost/webservices/soap/ws-user-account.php" \
-H "Content-Type: text/xml; charset=UTF-8" \
-H "SOAPAction: \"urn:ws-user-account#getUser\"" \
--data-binary @- <
   
   
      
         Jeremy
      
   

EOF

2. registerUser Operation

POST /webservices/soap/ws-user-account.php HTTP/1.1
Host: mutillidae.localhost
Content-Type: text/xml; charset=UTF-8
SOAPAction: "urn:ws-user-account#registerUser"

<soapenv:Envelope xmlns:soapenv="http://schemas.xmlsoap.org/soap/envelope/" 
                  xmlns:urn="urn:ws-user-account">
   <soapenv:Header/>
   <soapenv:Body>
      <urn:registerUser>
         <username>Joe</username>
         <password>Holly</password>
         <firstname>Joe</firstname>
         <lastname>Holly</lastname>
         <signature>Try Harder</signature>
      </urn:registerUser>
   </soapenv:Body>
</soapenv:Envelope>

cURL Command

curl -X POST "http://mutillidae.localhost/webservices/soap/ws-user-account.php" \
-H "Content-Type: text/xml; charset=UTF-8" \
-H "SOAPAction: \"urn:ws-user-account#registerUser\"" \
--data-binary @- <
   
   
      
         Joe
         Holly
         Joe
         Holly
         Try Harder
      
   

EOF
Troubleshooting Tips:
  • Ensure that the host mutillidae.localhost is properly configured in your environment.
  • Use the correct SOAPAction header for the operation you want to perform.
  • Ensure that the Content-Type is set to text/xml; charset=UTF-8.
